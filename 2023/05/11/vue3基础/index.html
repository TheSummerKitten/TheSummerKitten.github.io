2.setup选项

setup的写法和时机

与生命周期钩子类似,但是其实执行时间要比beforeCreate还要早,所以获取不到this(this是undefined),而且以后基本上不会再使用this

setup可以提供数据,也可以提供函数,但是提供的东西都要在return这个选项中返回,这样数据一旦过度偶就会不会维护,所以有一个setup语法糖 :

reactive和ref函数

reactive() :

作用是 接收对象类型数据的参数传入并返回一个响应式的对象

ref() :

作用是 接收对象类型数据 或 简单数据类型的参数传入并返回一个响应式的对象

computed组合式API

相对于vue2修改了写法

1.导入computed函数

2.执行函数 在回调参数中 return 基于响应式数据做计算的值,用变量接收

1.计算属性中不应该有”副作用”

比如修改DOM/异步请求

2.避免直接修改计算属性的值

特殊情况可以配置get/set

watch函数

作用: 侦听一个或者多个数据的变化,数据变化时立刻执行回调函数 , 数据源还可以是函数

两个额外参数: immediate(立刻执行) deep(深度监听) once(一次性执行)

生命周期钩子

选项式API 组合式API
beforeCreate/created setup
beforeMount onBeforeMount
mounted onMounted
beforeUpdate onBeforeUpdate
updated onUpdated
beforeUnmount onBeforeUnmount
unmounted onUnmounted

defineOptions

在setup语法糖中, 可以使用 编译宏 来处理与setup同级的属性 , 但是要处理组件(页面) 的其他属性比如 name 就需要回归原始的用法 —- 再添加一个script标签.这么做非常难受, 所以我们使用了defineOptions 宏. 用来定义Options API 的选项, 但是 props , emits , expose , slots 除外
